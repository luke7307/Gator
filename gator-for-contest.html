# 产品思考 · 今日灵感

灵感不是被整理出来的，它是被捕获的

📅 刚刚 ✍️ 灵感条目 🏷️ #产品 #笔记

当工具要求你先成为"笔记系统设计师"，灵感的第一秒就已经流失了。

Gator 的答案：打开即写，写完即走——它不是一个"笔记 App"，而是一个"灵感容器"。

核心原则

三个设计信念，决定了它的每一个像素

Gator 不是功能列表的堆砌，它是一套围绕"灵感第一秒"设计的约束

⚡

轻量

不注册、不登录、不云同步。首屏 < 100ms 加载，零摩擦启动。灵感的第一秒，不该被任何弹窗打断。

🏷️

可分类

emoji 图标 + 分类标签 + 颜色编码，每条灵感一眼可辨。信息不是被归档，而是被点亮。

🎨

视觉友好

卡片式布局、瀑布流、图片预览、弹性动效——以最舒适的方式呈现信息，让回顾变成一种享受。

灵感的困境

在 AI 能回答一切的时代，
真正稀缺的是你的问题

每一个灵感闪现的瞬间，如果不能被即时捕获，99% 都会永远消失

01

灵感半衰期只有 3 秒

你在地铁上看到一句共鸣的话，打开备忘录要 5 步：解锁 → 找 App → 新建 → 归类 → 输入。等你准备好，灵感已经走了。

02

整理的成本高于创造

大多数笔记工具先让你"设计系统"——命名、打标签、归类。创造的冲动，被整理的压力掐死在起跑线。

03

信息在无限膨胀，注意力在崩溃

AI 让信息获取零门槛，但真正属于你的只有那些触动过你的瞬间。当你试图保存一切，最终什么都留不住。

现阶段功能

一个能跑起来的 MVP，不是完美的空想

以下功能均已在桌面应用中实现，可直接打开实机应用体验

核心
💡

灵感快速捕获

Cmd+Opt+G 全局热键随时唤出窗口，3 秒记录一条灵感。打开即写，写完即走，零心智负担。

核心
🎨

6 类可视化内容

灵感 / 笔记 / Web Clip / 引用 / 文件 / 图片，每类都有专属的卡片样式和视觉编码，一眼区分。

分类
🏷️

emoji + 标签系统

每个文件夹一个专属 emoji + 主题色。分类不靠手动输入，而靠视觉直觉，回顾时如浏览自己的思维地图。

浏览
🖼️

瀑布流卡片视图

所有内容以卡片瀑布流的方式呈现，图片自动缩略图、长文字自适应高度，灵感不再埋没在文件夹深处。

存储
💾

本地优先存储

所有数据保存在本地 JSON 与 markdown 文件中，纯文本可读。不联网、不上传、不出设备。

体验
🔄

触控板惯性滑动

原生级别的滚动体验，惯性感、弹性动效与系统应用一致。写东西这件事，手感比功能更重要。

当前进度

项目的真实状态

这是一个从第一行代码开始就被真实使用的产品，以下是目前所处的阶段

🌱 一个被真实使用的 MVP

核心链路 "捕获 → 归类 → 回顾" 已打通，日常可稳定使用

✓

核心数据模型 已完成

6 类内容（灵感/笔记/Web Clip/引用/文件/图片）的统一存储结构，每个条目包含唯一 ID、emoji、主题色、分类标签、时间戳。

✓

桌面应用骨架 已完成

基于 Electron 的多窗口架构，支持 Cmd+Opt+G 全局热键快速唤起，主窗口 + 快速捕获窗口分离。

✓

瀑布流卡片浏览 已完成

Masonry 布局实现卡片自适应排列，图片自动缩略图、长文字自适应高度，支持触控板惯性滑动。

●

文件夹与分类系统 进行中

每个文件夹支持自定义 emoji + 主题色 + 描述，左侧边栏文件夹切换 + 快速筛选。正在完善：标签合并、智能文件夹。

○

搜索与检索 待实现

全文搜索、关键词高亮、按时间/类型/标签多维度筛选。灵感回顾时，搜索是最高频的入口。

未来规划

从灵感容器，到思维引擎

以下是接下来几个阶段的方向——这是一个会持续生长的工具

PHASE 1 · 近 3 个月

完善基础体验

  • 全文搜索 + 关键词高亮，让过去的灵感秒级可及
  • 跨窗口同步：主窗口 ↔ 快速捕获窗口数据实时同步
  • 导出能力：Markdown 批量导出、HTML 快照存档
  • 主题切换：浅色 / 深色 / 琥珀护眼模式
PHASE 2 · 年内

让灵感"活起来"

  • 时间线视图：按周/月回顾自己的灵感流，形成自我叙事
  • 关联条目：让相关的灵感彼此"认识"，从碎片走向脉络
  • AI 辅助：基于本地内容进行语义检索、自动打标签、灵感推荐
  • 多设备同步（完全本地化的方案，不依赖第三方云）
PHASE 3 · 长期

真正的"思维引擎"

  • 个人知识图谱：灵感之间的连接可视化
  • 创作工作台：将捕获的灵感一键转化为文章/方案大纲
  • 开放插件：让第三方开发者扩展捕获渠道
  • 从"保存想法"进化到"孕育思想"——让真正属于你的东西持续生长
