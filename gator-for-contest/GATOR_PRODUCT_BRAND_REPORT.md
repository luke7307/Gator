## 一、产品概述

### 1.1 产品名称与定位

**产品名称**：Gator  
**副标语**：信息效率工具箱  
**产品定位**：一款专注于"灵感快速捕获与管理"的跨平台笔记应用

### 1.2 核心理念

Gator 的核心不是"写笔记"，而是"捕捉灵感"。它解决一个核心问题：

> **当我有一个想法 / 灵感 / 知识点时，我如何以最少的操作把它留下来，并在以后能方便地找到它？**

**产品哲学三原则**：

1. **轻量** — 打开即用，不注册、不登录、不云同步（当前版本）。数据本地存储（localStorage）
2. **可分类** — 每条灵感都有 emoji 图标 + 分类标签，一眼可辨
3. **视觉友好** — 卡片式浏览、图片预览、瀑布流布局，信息以最舒适的方式呈现

### 1.3 产品名称语义（v2.0，已废弃短吻鳄联想）

**Gator** = 从 **Gather**（聚合、收集）简化而来

**语义体系**：

| 语义方向 | 含义 | 与产品关联 |
|---------|------|-----------|
| **Gather 聚合** | 收集散落的灵感、碎片、想法 | ✅ 核心功能：捕获和整理信息 |
| **Gateway 入口** | 你的灵感和知识的统一入口 | ✅ 打开即用的主界面 |
| **Generator 生成** | 从碎片化信息中生成价值 | ✅ 分类、搜索、回顾 |
| **Guardian 守护者** | 安全地保存你的每一条灵感 | ✅ 本地存储、不丢失 |

> **一句话版本**：Gator 是你灵感的聚合器 — 把每一个闪现的想法稳稳收进来。

**不再使用的语义**：
- ❌ 短吻鳄 / 鳄鱼 / 动物形象
- ❌ "咬住不放"、"捕食"等攻击性隐喻
- ❌ 绿色 / 土色等爬行动物关联色

### 1.4 目标用户

- 需要快速记录灵感 / 想法的个人用户
- 信息工作者（学生、研究员、内容创作者）
- 偏好本地存储、不希望数据上云的用户

## 三、核心交互流程

### 3.1 创建一条灵感

```
首页（空 / 有内容）
  ↓ 点击右下角 FAB (+)
FAB 旋转展开 → 弹窗输入
  ↓
输入标题 + emoji + 分类 + 正文（支持富文本）
  ↓
点 X 关闭（自动保存）→ 返回首页 → 卡片高亮泛光 2 秒 → "相关修改已保存" Toast
```

### 3.2 浏览与查看

```
首页卡片列表
  ↓ 点击卡片
进入预览模式（全屏弹窗，文字 contenteditable=false）
  ├─ 点击 emoji → 弹出 emoji 选择器 → 更换 → _previewModified = true
  ├─ 点击分类标签 → 弹出分类选择 → 更换 → _previewModified = true
  ├─ 点击任务 checkbox → 勾选 / 取消 → saveFullscreenChanges 检测
  └─ 双击空白区域 / 点右上角笔图标 → 进入编辑模式
  ↓ 点 X 关闭
保存变更（检查 _previewModified 和内容差异）→ 返回首页 → 增量更新单卡片（不重建整列表）
```

### 3.3 编辑一条灵感

```
预览模式 → 点笔图标
  ↓
进入全屏编辑器（contenteditable=true）
  ↓
编辑标题、正文、插入图片、格式化
  ↓
点 X 关闭 → 如有差异：弹窗确认"不保存 / 保存" → 返回首页 → 卡片高亮泛光
         如无差异：直接关闭
```

### 3.4 查看图片

```
卡片图片区域 / 预览模式图片
  ↓ 点击
图片预览查看器（全屏，高斯模糊背景）
  ├─ 底部玻璃工具栏：缩小 / 放大 / 全屏 / 旋转 / 下载
  ├─ 右上角：图片计数器 1/N
  ├─ 左上角：关闭按钮
  └─ 左右箭头：切换多图
  ↓ 点 X / 点空白 / ESC
关闭 → 恢复到之前视图
```

### 3.5 删除与恢复

```
卡片左滑 → 露出红色删除按钮
  ↓ 点击删除
弹出删除确认弹窗（垃圾桶 SVG 循环摇摆动画）
  ↓ 确认删除
卡片滑出消失 → Toast "已删除，30 天内可在回收站恢复"
  ↓
回收站 → 找到卡片 → 点击恢复 → 卡片回到首页并高亮泛光
```

**SVG 图标规范**：

- **风格** — 线性 outline 图标（stroke, not fill），简洁、现代
- **线条粗细** — stroke-width: 1.9 - 2.0
- **线条端点** — stroke-linecap: round
- **线条连接** — stroke-linejoin: round
- **viewBox** — 标准 24 × 24
- **颜色** — currentColor（通过父元素 color 属性控制）
- **不使用填充图标** — 产品整体风格偏轻盈，填充图标会显得过重

### 4.6 动效设计

Gator 的动效哲学：**克制、有弹性、有反馈感**

| 动效 | 参数 | 应用场景 |
|-----|------|---------|
| **btn-press** | `scale(0.92)`, 0.15s `cubic-bezier(0.4, 0, 0.2, 1)` | 所有可点击元素点击反馈 |
| **fadeIn** | opacity 0→1, 0.2s ease | 弹窗出现 |
| **delete-modal-in** | scale 0.9→1 + translateY 10→0, 0.3s `cubic-bezier(0.16, 1, 0.3, 1)` | 删除确认弹窗 |
| **fab-pulse** | box-shadow 呼吸, 2.5s ease-in-out infinite | FAB 按钮等待点击时 |
| **fab-plus-pop** | scale 1→1.25, 0.5s `cubic-bezier(0.34, 1.56, 0.64, 1)` | FAB hover 加号弹出 |
| **fab-icon-spin** | rotate 360°, 0.55s `cubic-bezier(0.4, 0, 0.2, 1)` | FAB 菜单展开图标旋转 |
| **pin-anim** | translateY + rotate, 弹性曲线 | 置顶 pin 针交互动画 |
| **highlight glow** | box-shadow + outline 主题色, 2s 淡出 | 编辑 / 恢复后卡片高亮 |
| **card-slide** | translateX -70px, 0.3s `cubic-bezier(0.32, 0.72, 0, 1)` | 卡片左滑删除 |
| **search-focus** | border-color + box-shadow 过渡 0.6s | 搜索框聚焦泛光 |
| **gear-spin** | rotate 180°, 0.5s `cubic-bezier(0.34, 1.56, 0.64, 1)` | 分类管理齿轮旋转 |

> 动效设计关键词：**弹性（elastic / spring）** → 使用 `cubic-bezier(0.34, 1.56, 0.64, 1)` 等超出 1.0 的曲线，产生"弹一下"的质感；**克制** → 不滥用动画，只在交互反馈和状态切换时使用。

### 7.2 Logo 设计指引（v2.1）

**核心概念**：Gather 聚合 — 信息效率工具的视觉入口

> Gator 的 Logo 不是一个"图标"，而是一个"入口"。它应当让人一眼感知到：**这里可以收拢和管理我的灵感碎片**。

**设计原则**（按优先级）：

| 原则 | 说明 | 技术表达 |
|-----|------|---------|
| **1. 圆形优先** | Logo 的外轮廓必须是完美圆形 | `border-radius: 9999px / viewBox="0 0 24 24"` |
| **2. 几何构成** | 全部由基本几何形（圆、弧、线段）组成，不使用任何具象图形 | 避免动物、人物、写实元素 |
| **3. 粗线条** | stroke-width ≥ 2.0，保证缩小到 24px 时仍清晰可辨 | `stroke-width: 2.0pt ~ 3.0pt` |
| **4. 双版本** | 同一 Logo 提供填充版（彩色背景 + 白色几何）和线稿版（彩色线条 + 透明背景） | 两套文件共用同一几何骨架 |
| **5. 少细节** | 不超过 3 个视觉元素（外轮廓 + 主体几何 + 0-1 个强调点） | 避免装饰性小元素 |

**色彩体系**：

| 版本 | 背景 | 主体色 | 辅助色 | 适用场景 |
|-----|------|--------|--------|---------|
| **填充版（彩色圆）** | 线性渐变 135° `#3B82F6 → #2563EB` | `#FFFFFF` 白色几何线条 | `#F97316` 橙色（内部光点，≤ 1 处） | App Icon、FAB 按钮、启动页 |
| **线稿版** | 透明 / 白色 | `#3B82F6` 蓝色 / `#374151` 深灰 | `#F97316` 橙色（可选） | 文档、品牌页、水印 |
| **单色版** | 透明 | `#FFFFFF` / `#1F2937` | 无 | 深色背景、灰度媒体 |

**比例与安全区**：

```
外轮廓:           完美圆形 1:1
内部安全区:       ≥ 10% 边距（几何元素不贴边）
FAB 内安全区:     ≥ 18% 边距（按钮与 Logo 之间留白）
与文字间距:       1× Logo 高度（Logo 与"Gator"之间不挤）
推荐绘制画布:     24×24 或 100×100 viewBox
推荐线条粗细:     2.0pt ~ 3.0pt（根据画布缩放）
```

**不建议出现的元素**（设计雷区 · 绝对避免）：

- ❌ 任何动物形象（鳄鱼、短吻鳄等）
- ❌ 写实插画、照片风格
- ❌ 3D 立体效果（阴影、高光、拟物）
- ❌ 尖角、锯齿、刺状轮廓（与产品圆润基调冲突）
- ❌ 超过 3 种颜色
- ❌ 小于 2.0pt 的细线（缩到小尺寸后消失）
- ❌ 装饰性元素（花纹、渐变文字、复杂字体）

**一句话设计目标**：一个蓝色的圆，里面有清晰的几何形状，能让人联想到"入口 / 聚合 / 添加灵感"，并且缩小到 24px 时还能看出是什么。

### 7.3 Logo 设计方向（4 个方向供选择）

#### 方向 A：G 圆 · 聚合环（推荐优先级：最高）

**概念**：一个不闭合的圆形 G，暗示"聚合"和"入口"

**视觉元素**：
- 外轮廓：完美圆形，与 FAB 按钮一致
- 主体：字母 G 的几何简化（一条粗弧线）
- 开口：右上角有一个缺口，暗示"收进来 / 入口"
- 内部：一个小圆点 / 发光点，代表"灵感"
- 风格：扁平几何，线条粗细一致

**版本建议**：
- 完整版（300px+）：G 圆弧 + 内部光点 + 可选橙色强调
- 简化版（FAB / 小尺寸）：只保留 G 圆弧，去掉内部光点
- 线稿版（文档中使用）：蓝色线条 + 橙色光点

**适用场景**：✅ App Icon · ✅ FAB 按钮内部 · ✅ 启动页 Logo · ✅ 小尺寸图标

#### 方向 B：圆点 · 灵感网格（推荐优先级：中）

**概念**：多个小圆点汇聚成一个整体，代表"聚合灵感碎片"

**视觉元素**：
- 中心：一个较大的实心圆（代表当前聚焦的灵感）
- 周围：3-5 个较小的圆点，向中心聚拢（代表收集的碎片）
- 整体布局：形成一个松散的 G 轮廓或同心圆
- 风格：点组成形、极简、有节奏感

**色彩方案**：
- 主色点：`#3B82F6`（蓝）
- 辅助色点：`#8B5CF6`（紫）、`#10B981`（绿）、`#F59E0B`（橙）
- 背景：白色或浅蓝色 `#EFF6FF`

**适用场景**：✅ 启动页 / 品牌页 · ✅ 空状态插图 · ✅ 社交媒体品牌图形

#### 方向 C：G + 加号（极简功能化，推荐优先级：高）

**概念**：字母 G 和加号 "+" 的几何融合，强调"添加灵感"这个核心动作

**视觉元素**：
- 主体：大写字母 G 的简化轮廓（圆形 + 横杠）
- 融合：G 的内部横杠同时也是一个 + 加号
- 或者：一个圆形背景中有一个白色 +，但 + 的笔画加粗后暗示 G 的形状
- 风格：极简几何、粗线条、高度符号化

**色彩方案**：
- 背景圆：蓝色渐变 `#3B82F6 → #2563EB`
- 前景符号：白色粗线条 +/G 融合体
- 线稿版：蓝色线条

**适用场景**：✅ FAB 按钮图标（高度贴合当前 FAB 的 + 加号）· ✅ App Icon · ✅ 小尺寸品牌标识

#### 方向 D：卡片流 · 瀑布流抽象（推荐优先级：低 — 仅用于插图）

**概念**：从产品的核心界面 — 卡片瀑布流 — 提取视觉特征

**视觉元素**：
- 2-3 个不同高度的圆角矩形（代表卡片）
- 错落排列，模拟瀑布流布局
- 顶部或中心有一个发光点（代表灵感灯泡）
- 最左侧或中心卡片用主题色填充，其余白色描边
- 整体外轮廓：收窄成一个圆形或方形边界

**色彩方案**：
- 主卡片：蓝色填充 `#3B82F6`，白色 emoji 💡
- 辅助卡片：白色填充 + 灰色描边 `#E5E7EB`
- 背景：浅蓝 `#EFF6FF` 或白色

**适用场景**：✅ 产品首页插图 · ✅ 启动页大图 · ✅ 文档 / 教程配图 · ⚠️ 不建议作为 App Icon

| **🟡 中** | 首页空状态插图 | 1 个 SVG 插图 |
| **🟡 中** | 品牌页 / 文档品牌标识系统 | 一套使用规范 |
| **🟢 低** | 回收站空状态插图 | 1 个 SVG 插图 |

## 十、设计雷区 · 避免事项

| ⚠️ 雷区 | 说明 | 替代方案 |
|--------|------|---------|
| **复杂插画** | 产品整体极简，复杂插画会显得突兀 | 使用扁平几何 + emoji 结合 |
| **尖角元素** | 产品 UI 全是圆角，尖角 Logo 不协调 | 大圆角（16px+）、圆形外轮廓 |
| **3D / 拟物** | 产品是扁平风格，3D 图标不匹配 | 扁平几何、线条风格 |
| **多种字体** | 产品只用 Inter/PingFang，避免装饰字体 | 保持 Inter 一种字体 |
| **动物形象** | 已废弃短吻鳄 / 鳄鱼语义 | 纯粹几何符号（G 圆、聚合、入口） |
| **高饱和撞色** | 产品使用柔和渐变，避免刺眼配色 | 主色 + 互补色（蓝 + 橙） |
| **过小的细节** | Logo 需要能缩到 24px 仍清晰 | 粗线条、大几何、少细节 |

**推荐做法**：几何构成 + 圆润线条 + 单一或双色 + 可缩放到 24px 仍清晰
