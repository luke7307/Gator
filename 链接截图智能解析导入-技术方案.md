# 链接/截图智能解析导入 — 技术方案

## 一、整体架构

```
用户输入（链接/截图/剪贴板）
    │
    ├─ 链接 ──→ 平台识别 ──→ 内容抓取 ──→ 结构化数据 ──→ 模板渲染
    │
    ├─ 截图 ──→ OCR 识别 ──→ 文本提取 ──→ 模板渲染
    │
    └─ 纯文本 ──→ 智能解析（URL提取/关键词） ──→ 模板渲染
```

function detectPlatform(url) {
  for (const rule of PLATFORM_RULES) {
    if (rule.patterns.some(p => p.test(url))) return rule;
  }
  return { name: '网页', icon: 'link', color: '#6B7280' };
}

### 2. 内容抓取 — 三层方案（由简到深）

#### 方案 A：Open Graph / Meta 标签解析（最轻量，推荐先做）

绝大多数平台页面都有 OG 标签，Electron 环境可以直接用隐藏 BrowserWindow 加载页面后提取：

```javascript
// Electron 主进程
const { BrowserWindow } = require('electron');

async function fetchLinkMeta(url) {
  const win = new BrowserWindow({
    width: 800, height: 600,
    show: false,
    webPreferences: { nodeIntegration: false }
  });

  try {
    await win.loadURL(url, { userAgent: '...' });
    // 等待关键内容渲染
    await new Promise(r => setTimeout(r, 2000));

    const meta = await win.webContents.executeJavaScript(`
      (function() {
        const getMeta = (name) =>
          document.querySelector('meta[property="${name}"]')?.content ||
          document.querySelector('meta[name="${name}"]')?.content || '';

        const getImages = () => {
          const ogImg = getMeta('og:image');
          const twitterImg = getMeta('twitter:image');
          // 小红书/微博的图片列表
          const images = [...document.querySelectorAll('img')]
            .map(img => img.src)
            .filter(src => src && !src.includes('avatar') && src.startsWith('http'));
          return [...new Set([ogImg, twitterImg, ...images].filter(Boolean))];
        };

        return {
          title: getMeta('og:title') || document.title || '',
          description: getMeta('og:description') || getMeta('description') || '',
          images: getImages().slice(0, 9),
          siteName: getMeta('og:site_name') || '',
          type: getMeta('og:type') || 'article',
          video: getMeta('og:video') || '',
          publishTime: getMeta('article:published_time') || '',
          author: getMeta('author') || getMeta('article:author') || '',
        };
      })()
    `);

    return meta;
  } finally {
    win.destroy();
  }
}
```

**优点**：无需第三方服务，本地完成，隐私安全
**缺点**：部分平台（抖音、小红书）需要登录或反爬，OG 标签可能不完整

#### 方案 B：第三方解析 API（最省心，推荐生产环境）

| 服务 | 特点 | 价格 |
|------|------|------|
| **Jina Reader** (`r.jina.ai`) | 开源，返回 Markdown | 免费/按量 |
| **microlink.io** | 结构化 OG + 截图 | 免费额度 |
| **linkpreview.net** | 纯 OG 解析 | 免费 60次/时 |
| **iframe.ly** | 支持 oEmbed | 免费额度 |

Jina Reader 方案（推荐，免费且返回 Markdown 正文）：

```javascript
async function fetchByJina(url) {
  const resp = await fetch(`https://r.jina.ai/${url}`, {
    headers: { 'Accept': 'text/markdown' }
  });
  const markdown = await resp.text();
  return parseMarkdownToStructured(markdown);
}
```

#### 方案 C：平台专用 API/爬虫（最完整，后续扩展）

```javascript
// 各平台专用解析器
const PARSERS = {
  weibo: {
    // 微博 m.weibo.cn 的 API，返回 JSON
    async parse(url) {
      const id = url.match(/status\/(\d+)/)?.[1] ||
                 url.match(/detail\/(\d+)/)?.[1];
      if (!id) return null;
      const resp = await fetch(`https://m.weibo.cn/statuses/show?id=${id}`);
      const data = await resp.json();
      const card = data.data;
      return {
        title: card.status_title || '',
        content: card.text,  // HTML
        author: card.user?.screen_name,
        avatar: card.user?.profile_image_url,
        images: (card.pics || []).map(p => p.large?.url || p.url),
        video: card.page_info?.media_info?.stream_url_hd,
        reposts: card.reposts_count,
        comments: card.comments_count,
        likes: card.attitudes_count,
        publishTime: card.created_at,
      };
    }
  },

  bilibili: {
    async parse(url) {
      // b23.tv 短链先跟随重定向
      if (url.includes('b23.tv')) {
        const resp = await fetch(url, { redirect: 'follow' });
        url = resp.url;
      }
      const bvid = url.match(/BV[\w]+/)?.[0];
      if (!bvid) return null;
      const resp = await fetch(
        `https://api.bilibili.com/x/web-interface/view?bvid=${bvid}`
      );
      const data = (await resp.json()).data;
      return {
        title: data.title,
        description: data.desc,
        author: data.owner?.name,
        avatar: data.owner?.face,
        cover: data.pic,
        video: `https://www.bilibili.com/video/${bvid}`,
        duration: data.duration,
        views: data.stat?.view,
        danmakus: data.stat?.danmaku,
        likes: data.stat?.like,
        publishTime: new Date(data.pubdate * 1000).toISOString(),
        tags: (data.tag || '').split(','),
      };
    }
  },

  xiaohongshu: {
    // 小红书反爬严格，推荐用 Jina 或自建服务
    async parse(url) {
      // 短链 xhslink.com 先解析重定向
      // 然后用 Jina 获取正文
      return fetchByJina(url);
    }
  },

  douyin: {
    // 抖音同理，反爬严格
    async parse(url) {
      return fetchByJina(url);
    }
  },

  wechat: {
    // 公众号文章 OG 标签较完整，方案 A 即可
    async parse(url) {
      return fetchLinkMeta(url);
    }
  }
};
```

## 三、剪贴板监听（自动识别）

```javascript
// Electron 渲染进程
document.addEventListener('paste', (e) => {
  const text = e.clipboardData.getData('text/plain');
  const image = e.clipboardData.files[0];

  // 优先检测图片（截图）
  if (image && image.type.startsWith('image/')) {
    handleScreenshotPaste(image);
    return;
  }

  // 检测链接
  const urlRegex = /https?:\/\/[^\s<>"']+/g;
  const urls = text.match(urlRegex);
  if (urls) {
    urls.forEach(url => handleLinkPaste(url));
    return;
  }

  // 纯文本 → 直接填入导入框
  handleTextPaste(text);
});

// 后台监听剪贴板变化（可选）
let lastClipboard = '';
setInterval(async () => {
  const text = await navigator.clipboard.readText().catch(() => '');
  if (text && text !== lastClipboard) {
    lastClipboard = text;
    const urls = text.match(/https?:\/\/[^\s<>"']+/g);
    if (urls) showLinkImportTip(urls); // 提示用户"检测到链接，是否导入"
  }
}, 2000);
```

## 六、推荐实施路线

| 阶段 | 内容 | 方案 |
|------|------|------|
| **P0** | 链接识别 + OG 解析 + 卡片模板 | 方案A（本地 BrowserWindow） |
| **P1** | 剪贴板监听 + 自动提示 | paste 事件 + 轮询 |
| **P2** | 平台专用解析（微博/B站） | 方案C（公开API） |
| **P3** | 截图 OCR 导入 | Tesseract.js / Vision |
| **P4** | Jina Reader 通用正文提取 | 方案B |
| **P5** | AI 总结/标签/分类 | LLM API |

**P0 就能覆盖 80% 的场景**：用户粘贴链接 → 识别平台 → 抓取 OG 标签 → 渲染卡片。微信公众号、B站、普通网页的 OG 标签都比较完整，微博和小红书可能需要 P2/P4 补充。
